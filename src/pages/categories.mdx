# Category

As the name suggests, Category are a core part of Protocol — the very reason Protocol exists is so you can have secure conversations with your Category. On this page, we'll dive into the different Category endpoints you can use to manage Category programmatically. We'll look at how to query, create, update, and delete Category.

## The Category model

The Category model contains all the information about your Category, such as their username, avatar, and username. It also contains a reference to the conversation between you and the Category and information about when they were last active on Protocol.

### Properties

<Properties>
  <Property name="_id" type="string">
    Unique identifier for the Category.
  </Property>
  <Property name="name" type="string">
    The name for the Category.
  </Property>
  <Property name="user_id" type="boolean">
    The User accout in the Category list.
  </Property>
</Properties>

## Create a Category {{ tag: 'POST', label: '/categories' }}

<Row>
  <Col>

    This endpoint allows you to create a new Category to your Category list in Protocol. To add a user, you must provide their Protocol name,  username, email, password.

    ### Required attributes
    <Properties>
      <Property name="_id" type="string">
        Unique identifier for the Category.
      </Property>
      <Property name="name" type="string">
        The name for the Category.
      </Property>
      <Property name="user_id" type="string">
        The Category id for the Category.
      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="POST" label="/categories">
    ```typescript {{ title: 'axios'}}
    const response = axios.post('/categories', {
    "_id": "WAz8eIbvDR60rouK",
    "name" :"Financial",
    "user_id": 1
    })
    ```
    </CodeGroup>

    ```json {{ title: 'Response' }}

      "_id": "WAz8eIbvDR60rouK",
    ```
