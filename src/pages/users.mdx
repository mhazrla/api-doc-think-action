# Users

As the name suggests, users are a core part of Protocol — the very reason Protocol exists is so you can have secure conversations with your users. On this page, we'll dive into the different user endpoints you can use to manage users programmatically. We'll look at how to query, create, update, and delete users.

## The user model

The user model contains all the information about your users, such as their username, avatar, and username. It also contains a reference to the conversation between you and the user and information about when they were last active on Protocol.

### Properties

<Properties>
  <Property name="_id" type="string">
    Unique identifier for the user.
  </Property>
  <Property name="name" type="string">
    The name for the user.
  </Property>
  <Property name="username" type="string">
    The username for the user.
  </Property>
  <Property name="email" type="string">
    The user email in the user list.
  </Property>
  <Property name="password" type="string">
    The user password in the user list.
  </Property>
  <Property name="bio" type="string">
    The user bio in the user list.
  </Property>
  <Property name="acc_type" type="boolean">
    The user accout type in the user list.
  </Property>
  <Property name="photo" type="string">
    The user photo in the user list.
  </Property>
</Properties>

## Update a user {{ tag: 'PUT', label: '/change-profile' }}

<Row>
  <Col>

    This endpoint allows you to perform an update on a user. Currently, the only attribute that can be updated on users is the `name`,`username`,`email`,`password` attribute which controls how a user appears in your user list in Protocol.

    ### Optional attributes

    <Properties>
      <Property name="name" type="string">
        The user name in the user list. By default, this is just the name.
      </Property>
      <Property name="username" type="string">
        The user username in the user list. By default, this is just the username.
      </Property>
      <Property name="email" type="string">
        The user email in the user list. By default, this is just the email.
      </Property>
      <Property name="bio" type="string">
        The user bio in the user list. By default, this is just the bio.
      </Property>
      <Property name="photo" type="image">
        The user photo in the user list. By default, this is just the photo.
      </Property>
    </Properties>

  </Col>
  <Col sticky>
     <CodeGroup title="Request" tag="PUT" label="/change-profile">
    ```typescript {{ title: 'axios'}}
    const response = axios.put('/change-profile', {
      "name": "John Doe",
      "username": "johndoe",
      "email": "johndoe@gmail.com",
      "bio": "Bio Example",
      "acc_type": 1,
    ```

## Delete a user {{ tag: 'DELETE', label: '/users/:user_id' }}

<Row>
  <Col>

    This endpoint allows you to delete users from your user list in Protocol. Note: This will also delete your conversation with the given user.

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="DELETE" label="/users/WAz8eIbvDR60rouK">

    ```bash {{ title: 'cURL' }}
    curl -X DELETE https://api.protocol.chat/users/WAz8eIbvDR60rouK \
      -H "Authorization: Bearer {token}"
    ```

    ```js
    import ApiClient from '@example/protocol-api'

    const client = new ApiClient(token)

    await client.users.delete('WAz8eIbvDR60rouK')
    ```

